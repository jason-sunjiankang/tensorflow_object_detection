TensorFlow:Object_Detection_API在Windows10上的配置



在models-master/research/object_detection(tensorflow object detection api)路径下,执行下面的相关命令：
（！！！！！！直接粘贴到cmd容易错行，要检查好！！！！！！）
1.训练：
python3 train.py \--logtostderr \--pipeline_config_path=/home/sun/tensorflow/project1_object_detection/object_detection/data/ssd_mobilenet_v1_raccoon.config \--train_dir=/home/sun/tensorflow/project1_object_detection/object_detection/model/train_model

2.测试：
python3 eval.py \--logtostderr \--pipeline_config_path=/home/sun/tensorflow/project1_object_detection/object_detection/data/ssd_mobilenet_v1_raccoon.config \--checkpoint_dir=/home/sun/tensorflow/project1_object_detection/object_detection/model/train_model \--eval_dir=/home/sun/tensorflow/project1_object_detection/object_detection/model/eval

运行测试时会提示 No module named 'pycocotools'或No module named 'object_detection'，原因是没有按照官方进行环境配置：在tensorflow/models/research/下执行export PYTHONPATH=$PYTHONPATH:`pwd`:`pwd`/slim
（如果关闭终端窗口，则必须重新执行此操作！！！！！！！）


3.将检查点文件导出为冻结的模型文件：
python3 export_inference_graph.py \--pipeline_config_path=/home/sun/tensorflow/project1_object_detection/object_detection/data/ssd_mobilenet_v1_raccoon.config \--trained_checkpoint_prefix=/home/sun/tensorflow/project1_object_detection/object_detection/model/train_model/model.ckpt-60 \--output_directory=/home/sun/tensorflow/project1_object_detection/object_detection/model/train_model
